What is version control?
This repo is just a test repo for me to get used to the GitHub interface and learn the terminology that developers use like Commits. I answer some questions below.

Why do developers use commits?
Version control is the practice of tracking and managing changes made to a software project. It acts as a database for modifications, allowing developers to work on the same project simultaneously without interfering with each others' work.

A commit is a saved version of the repository after changes are made. Developers are able to revert back to specific, stable versions of a repository using commits, allowing them to safely experiment with the code without the risk of losing the original stable code.
